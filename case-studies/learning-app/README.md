# Learning App – Improving User Retention

## Problem

Many learning apps struggle with user drop-off after initial onboarding. Users sign up but fail to build long-term engagement habits.

---

## Users

- New learners exploring a language learning app
- Users with low motivation after first few sessions

---

## Key Insight

Retention is less about content volume and more about early habit formation and perceived progress.

---

## Proposed Solution

### 1. Onboarding Flow Redesign

- Reduce initial friction
- Introduce immediate “small win” activity

### 2. Early Progress Feedback

- Visual progress indicators
- Micro-rewards after first actions

### 3. Habit Loop Design

- Daily reminder structure
- Lightweight session entry points

---

## Simple Flow

Sign-up → Quick win task → Progress feedback → Daily return trigger

---

## Prioritisation Rationale

Focus is placed on early engagement rather than feature expansion because user dropout occurs within the first 1–3 sessions.

---

## Outcome (Expected)

- Improved first-session completion rate
- Increased Day 3 / Day 7 retention
- Higher activation-to-habit conversion

---

## Notes

This case study is a conceptual product exercise demonstrating structured thinking in user retention and flow design.
